# 游戏：策划案- 都市尸潮初版DeepThink



# 策划案提示词

游戏制作人（Producer）兼主策划（Lead Designer）。我现在需要你担任游戏主策和主程身份，和我一起完成这个计划。

我已经详细评估了GitHub 仓库 `CyberPoincare/Zombie-Crisis` 以及线上试玩版。同时，为了更深入地挖掘团队的技术潜力，我也顺带查看了你 GitHub 上的其他项目（如 `MNCARender`）。

### **第一部分：项目诊断 (Status Check)**

**1. 现状分析：**
目前的 `Zombie-Crisis` 是一个 **"技术原型 (Tech Demo)"**。

- **核心机制**：它验证了基于 Web 的网格（Grid）移动、基础寻路和简单的判定逻辑。
- **问题所在**：
    - **缺乏“爽点” (Game Juice)**：目前的战斗是“你拍我一掌，我踢你一脚”的回合制/即时制混合，缺乏视觉冲击力。
    - **没有闭环 (No Loop)**：玩家玩 5 分钟后就会问“然后呢？”，没有资源积累，没有成长的验证。
    - **商业化为零**：目前的形态无法支撑买量成本（UA Cost），缺乏付费深度。

**2. 挖掘出的核心优势 (Your Secret Weapon)：**
我注意到你对 **Cellular Automata (细胞自动机/流体模拟)** 有深入研究（MNCARender 项目）。

- **机会点**：这是你最大的商业武器！目前市面上的丧尸手游大多是简单的模型堆叠。如果你能将 **MNCA 算法**引入到这款游戏中，实现**“像液体一样流动的万尸潮”**（类似《World War Z》或《Days Gone》的简化版），这将是极其炸裂的买量素材，能极大地降低获客成本。

### **第二部分：《Zombie Crisis: Hive Mind》 商业化改版策划案**

为了在红海市场（如《State of Survival》、《Last War》）中突围，我们不能做传统的换皮。我给你的定位是：**“带有视觉奇观的 Hybrid SLG（混合策略类）”**。

### **1. 产品定位 (Positioning)**

- **核心融合**：**Roguelite 塔防（副玩法/吸量）** + **4X COK-like 经营（主玩法/变现）**。
- **一句话概念**：用“流体尸潮”的视觉奇观吸引玩家，用“重建文明”的数值体系留住玩家。
- **目标用户**：25-45岁男性，泛策略用户，喜欢末日题材但厌倦了无脑点点点的玩家。

### **2. 核心玩法重构 (Gameplay Redesign)**

**A. 战斗层 (The Hook)：基于网格的“流体塔防”**

- **现有痛点**：现在的单位移动太死板。
- **改版方案**：
    - **核心机制**：玩家不再控制单一主角满图跑，而是**“据守”**。
    - **尸潮逻辑**：利用你的 MNCA 算法，丧尸不再是一个个独立寻路的个体，而是一股“黑色的水流”。它们会填满沟壑，会堆叠爬墙。
    - **玩家操作**：在网格上部署“英雄”和“设施”。
        - *英雄*：作为强力单点输出（如：加特林机枪手）。
        - *设施*：改变地形（如：放置电网墙阻挡尸潮流动，挖掘壕沟引导尸潮走向）。
    - **Roguelite 元素**：每守住一波，三选一强化（Perks），例如“子弹带电”、“增加无人机”、“路障血量翻倍”。

**B. 经营层 (The Loop)：避难所重建**

- **迷雾清理**：进入游戏时，基地 90% 被迷雾覆盖。玩家需要点击废墟区域，消耗体力进入“战斗层”清理尸潮，胜利后解锁该地块。
- **自由城建**：解锁的地块用于建造资源建筑（农场、油井）和功能建筑（兵营、科研所）。
- **挂机收益**：标准的 AFK 收益，保证玩家每隔 4-8 小时回流一次“收菜”。

**C. 战略层 (The Endgame)：联盟与大地图**

- **地缘政治**：玩家必须加入联盟。联盟领地在世界地图上通过六边形（Hex）铺设连接。
- **世界BOSS**：全服合力攻打“母巢”（超巨型 MNCA 聚合体）。
- **GVG**：争夺中心城市，获胜联盟获得全服税收权。

### **3. 商业化埋点 (Monetization)**

作为制作人，我设计的付费点必须确保 LTV (生命周期价值) 足够高。

**3.1 英雄系统 (Gacha - 核心收入)**

- 不要卖数值，要卖**“机制”**。
- **R级英雄**：基础步枪兵，数值怪。
- **SSR级英雄（付费点）**：
    - *“急冻人”*：技能可以冻结一片区域的“尸流”，改变尸潮路径。
    - *“工程师”*：可以瞬间建造 3 个临时炮台。
- **养成**：吃同名卡升星（深坑）。

**3.2 战斗通行证 (Battle Pass)**

- **生存手册**：
    - 免费版：给基础资源。
    - 高级版 ($9.99)：给限定皮肤、SSR 英雄碎片、加速道具。
- *设计目的*：通过高性价比转化小 R 玩家。

**3.3 资源货币化**

- **加速 (Speed-ups)**：SLG 的硬通货。建筑升级、造兵、科技研究都需要时间，付费买时间。
- **第二队列**：$4.99 永久解锁“第二建筑工”，这是必买项。

### **4. 开发落地路线图 (Execution Roadmap)**

基于 GitHub 仓库现状，我建议分三步走：

**Phase 1: 技术验证与 MVP (1.5个月)**

- **目标**：把 MNCA 尸潮跑通，结合现有的 Grid 射击，做一个好玩的单机 Demo。
- **核心任务**：
    1. **算法移植**：把 `MNCARender` 的流体逻辑简化，适配到 WebGL/Canvas。
    2. **数值重构**：抛弃一击必杀，引入 HP/DPS 模型。
    3. **视觉反馈**：加入简单的 Screen Shake（震屏）和伤害飘字。
    - *里程碑：发布到 Itch.io 测试核心玩法反馈。*

**Phase 2: 系统搭建 (2个月)**

- **目标**：接入 SLG 循环，让游戏能“玩下去”。
- **核心任务**：
    1. **城建 UI**：实现简单的拖拽建造、建筑升级逻辑。
    2. **数据存档**：接入后端（PlayFab 或 Firebase），保存玩家数据。
    3. **抽卡系统**：完成英雄池和概率逻辑。

**Phase 3: 商业化与上线 (2-3个月)**

- **目标**：接入广告和支付，准备买量。
- **核心任务**：
    1. **新手引导 (FTUE)**：这是生死线！前 15 分钟必须手把手教玩家：清理僵尸 -> 修墙 -> 抽卡。
    2. **SDK 接入**：支付、广告、统计埋点。

### **5. 给开发者的特别建议 (Producer's Note)**

1. **美术风格 (Art Style)**：
    - 目前的“程序员美术”无法商业化。建议采用 **Low-Poly（低多边形）风格**（参考 *Tomb of the Mask* 或 *Polytopia*）。这种风格资源便宜，且在这个赛道（僵尸）里做出来会很干净、独特，还能掩盖流体丧尸的细节瑕疵。
2. **技术栈警告**：
    - 如果目标是 App Store/Google Play，单纯的 Web 技术栈（React/Vue）可能会有性能瓶颈（特别是做尸潮时）。建议使用 **Cocos Creator** 或 **Unity** 进行重构，或者使用 WebGL 原生开发并打包成 App。
3. **配置驱动**：
    - 请立刻将代码中的数值（伤害、血量、掉率）解耦，建立 JSON/CSV 配置文件。商业游戏的调整频率极高，不能每次改数值都重新编译代码。

这是一个充满野心的计划，但你有独特的技术基因（MNCA）让它从平庸的丧尸游戏中脱颖而出。我们先从 **Phase 1 的流体尸潮 Demo** 开始吧！

# 游戏策划案：《Project: Z-Tide (代号：尸潮)》

## 1. 立项综述 (Executive Summary)

- **游戏类型**：**Hybrid SLG (混合策略)** = **流体塔防 (Roguelite)** + **合成经营 (Merge-Building)**。
- **对标竞品**：玩法结构参考《Top War (口袋奇兵)》的合成模式，战斗表现参考《World War Z》的尸潮堆叠。
- **一句话概念**：利用**流体算法**呈现万尸同屏的视觉奇观，结合**合成玩法**的爽快感，打造一款“看着爽、玩着轻、花钱深”的商业手游。
- **目标用户**：25-40岁男性，泛策略/塔防用户，喜欢末日题材但厌倦了换皮游戏的玩家。

---

## 2. 核心战斗：流体塔防 (The Battle Loop)

我们将原版仓库中的“单兵网格射击”彻底改造为**“上帝视角的战术塔防”**。

### 2.1 核心机制：流体场 (Flow Field)

利用你的 MNCA 技术，僵尸不再是独立的寻路个体，而是遵循流体力学的粒子。

- **流体寻路**：放弃 A* 寻路。地图生成一个指向基地核心的“向量场”。僵尸顺着向量流动。
- **物理特性**：
    - **压力与堆叠**：当网格密度达到 100% 时，僵尸会像水一样“溢出”到相邻网格。
    - **填坑逻辑**：遇到沟壑地形，前排僵尸自动死亡并填充高度，后排僵尸踩着尸体通过。
    - **漫灌**：如果高墙阻挡了尸潮，后方压力变大，尸潮会像洪水漫堤一样翻过墙壁。

### 2.2 玩家操作：地形与英雄

战斗不再是控制主角跑位，而是**“治水”**（截断、引流、蒸发）。

- **布阵阶段**：在 Grid 网格上布置“阻流墙”、“机枪塔”、“减速带”。
- **英雄单位**：作为超级塔防单位部署。
    - *“急冻人”*：技能是向尸潮最密集的区域投掷液氮，将流动的黑水瞬间冻结成**固体冰墙**，改变后续尸潮的流向。
- **Roguelite 成长**：每防守住一波（Wave），触发三选一（例如：*“子弹附带燃烧” / “墙体被破坏时爆炸” / “增加无人机支援”*）。

---

## 3. 外围系统：废土重建 (The Meta Game)

为了解决“玩完一局就走”的问题，必须引入长线养成。为了降低传统 SLG 的上手门槛，采用**“合成 (Merge)”**玩法。

### 3.1 基地建设 (Merge & Build)

- **视觉包装**：一个俯视角的废墟城市网格。
- **合成升级**：
    - **操作**：拖拽两个“1级兵营”重叠 -> 变成“2级兵营”。拖拽两个“1级机枪塔” -> 变成“2级机枪塔”。
    - **优势**：这种“拖拽即升级”的爽快感非常符合移动端用户习惯，比枯燥的“等待倒计时”更有吸引力。
- **迷雾探索**：
    - 基地周围被迷雾笼罩。
    - 点击迷雾地块 -> 消耗体力 -> 进入**流体塔防战斗**。
    - 胜利 -> 迷雾消散 -> 地块解锁（可用于摆放更多建筑）。

### 3.2 英雄系统 (Gacha RPG) - 核心付费点

- **SSR 英雄设计**：
    - **大坝 (The Dam)**：坦克位，放置后自带一个强力力场，能阻挡 5000 单位的流体压力。
    - **火神 (Vulcan)**：输出位，扇形喷火，对“粘液型”丧尸造成 300% 伤害。
- **养成**：升级、升星（吃同名卡）、装备。

---

## 4. 商业化设计 (Monetization)

作为制作人，我设计的付费点必须确保 LTV (生命周期价值) 足够高。

### 4.1 混合变现模型

1. **IAP (内购)**：
    - **首充礼包 ($0.99)**：送一把“无限弹药加特林”或 SSR 英雄。**这是转化率最高的点。**
    - **英雄抽卡**：设置保底池（如 80 抽必出 UP 角色）。
    - **第二队列**：$4.99 永久解锁“双倍合成队列”或“自动合成助手”。
2. **IAA (广告)**：
    - **绝境复活**：基地血量归零时，看广告复活并清屏 50%。
    - **三选一刷新**：Roguelite 选 buff 时，看广告刷新选项。
    - **离线收益翻倍**：上线收菜时，看广告获得 200% 资源。

### 4.2 痛点销售 (Pain Point Sales)

- **设计陷阱**：在第 10 关设计超高密度的尸潮，普通路障瞬间被冲垮。
- **解决方案**：死亡结算界面直接弹出——“限时特惠：高爆地雷包 ($1.99)”，专门克制密集尸潮。

## 5. 开发执行路线图 (Roadmap)

基于 GitHub 仓库现状，建议分三步走：

### **Phase 1: 技术验证与 MVP (1.5个月)**

- **目标**：把 MNCA 尸潮跑通，结合现有的 Grid 射击，做一个好玩的单机 Demo。
- **关键任务**：
    1. **技术栈迁移**：目前的 Web 前端栈（React/Vue）做尸潮性能会是瓶颈。建议使用 **Cocos Creator (TypeScript)** 或 **Unity**，或者使用纯 **WebGL (Three.js/Pixi.js)** 重写渲染层。
    2. **流场算法**：实现 2000 个红色粒子像水流一样绕过障碍物。
    3. **核心战斗**：实现“放置墙壁 -> 尸潮分流 -> 放置炮台 -> 击杀”的闭环。

### **Phase 2: 系统搭建 (2个月)**

- **目标**：接入合成玩法和数值系统。
- **关键任务**：
    1. **城建逻辑**：实现拖拽合成建筑的代码逻辑。
    2. **数据存档**：接入后端（PlayFab 或 Firebase），保存玩家数据。
    3. **配置驱动**：建立 JSON/CSV 配置表，将怪物血量、伤害等数值从代码中解耦。

### **Phase 3: 商业化与打磨 (2-3个月)**

- **目标**：接入广告和支付，准备上线。
- **关键任务**：
    1. **新手引导 (FTUE)**：前 10 分钟手把手教玩家：清理僵尸 -> 合成兵营 -> 抽卡。
    2. **SDK 接入**：支付、广告、统计埋点。
    3. **美术迭代**：建议采用 **Low-Poly (低多边形) + 霓虹特效** 风格。这种风格资源便宜，且高对比度的颜色能最大化“流体”的视觉冲击力。

## 6. 制作人的一点建议 (Actionable Advice)

1. **立刻停止**在现有 DOM/Canvas 代码上修补细节。目前的架构无法支撑“万尸同屏”。
2. **建立技术壁垒**：你的核心竞争力是 **MNCA 算法**。请先写一个 Demo，只渲染红点（僵尸）和蓝块（墙），如果能做到在手机浏览器上跑 60 帧且有流体感，你就成功了一半。
3. **美术取巧**：不要做写实丧尸（容易穿模且费资源）。把丧尸做成红色的“胶囊体”或“方块”，死亡时爆出一滩红色液体粒子。这种“抽象暴力美学”反而更吸量。
